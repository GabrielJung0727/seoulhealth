## 2. 예상 월간 운영 비용 (GCP 기준)

> ⚠️ **주의**: 아래는 GCP(Google Cloud Platform)로 운영할 때의 **예상치**입니다. 실제 트래픽과 사용량에 따라 달라집니다. 현재 코드베이스는 Vercel(프론트) + Railway(백엔드) 기본 세팅이지만, GCP로 이전 시 기준으로 계산했습니다.

### 2-1. 사용하는 GCP 서비스 구성

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│  [사용자]                                                │
│     ↓                                                   │
│  [Cloud DNS] — 도메인 seoulinkhealth.com 연결          │
│     ↓                                                   │
│  [Firebase Hosting] — 프론트엔드(React) 정적 파일 배포  │
│     ↓ API 호출                                          │
│  [Cloud Run] — 백엔드(Node.js/Express) 서버리스 컨테이너│
│     ↓                                                   │
│  [Cloud SQL] — PostgreSQL 데이터베이스                  │
│                                                         │
│  [SendGrid (외부)] — 이메일 발송 (SMTP)                 │
│  [Cloud Storage] — (선택) 파일/이미지 저장              │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 2-3. 비용 절약 팁

- ✅ **Cloud Run**은 "요청이 없을 때 0개 인스턴스" 설정으로 유휴 비용 제거 (단, 최초 응답이 1~2초 느려질 수 있음)
- ✅ **Cloud SQL**은 시나리오 A에서는 `db-f1-micro`로 충분. 데이터 100MB 이하면 과도한 업그레이드 불필요
- ✅ **Firebase Hosting 무료 티어**(10GB 저장 + 360MB/일 전송)가 초기엔 충분
- ✅ **SendGrid 무료 티어**(100통/일)만으로도 초기 운영 가능
- ⚠️ 주의: Cloud SQL 인스턴스는 사용 안 해도 상시 과금됨 — 개발 중단 시 **반드시 중지**

> 💡 **요약**: 런칭 직후 안정화 단계에서는 **월 약 ₩3~5만 원**, 본격 운영 시 **월 약 ₩15~25만 원**, 대규모 트래픽 시 **월 약 ₩50~80만 원** 선으로 예상하시면 됩니다.

## 3. Login 초보자 가이드

### 3-1. 로그인이란?

**로그인(Login) = "나는 이 사이트의 회원이에요. 제 이름으로 들어갈게요."라고 증명하는 절차**입니다.
호텔 체크인과 같습니다. 예약자 이름과 신분증을 보여주면 내 방 키를 받듯, 이메일/비밀번호를 입력하면 **내 전용 공간(Dashboard)**에 들어갈 수 있습니다.

### 3-2. SEOULINKHEALTH에는 두 가지 로그인이 있어요

| 구분 | 경로 | 누가 씀? | 인증 방식 |
|------|------|----------|-----------|
| **관리자 로그인** | `/login` | 사이트 운영자(이재원 박사님, 내부 담당자) | 비밀번호 1개 |
| **회사(기업) 로그인** | `/company/login` | K-Health 자문을 의뢰하는 기업 회원 | 이메일 + 비밀번호 + 6자리 OTP |

### 3-3. 관리자 로그인 (가장 간단)

1. 주소창에 `…/login` 입력
2. **비밀번호만** 입력하고 `Sign In` 클릭
3. 바로 관리자 Dashboard(`/admin`)로 이동

> 🔐 비밀번호는 서버의 환경변수(`ADMIN_PASSWORD`)로 저장되며, 평문으로 노출되지 않습니다.

### 3-4. 회사 로그인 (2단계 인증)

은행 앱처럼 **이중 확인**을 합니다. 이메일과 비밀번호가 맞아도 **등록된 이메일로 6자리 코드**가 발송되어야 완전히 로그인됩니다.

```
[1단계]  이메일 + 비밀번호 입력
         ↓
[자동]  등록된 이메일로 6자리 숫자 코드 발송
         ↓
[2단계]  받은 6자리 코드 입력
         ↓
[성공]  Company Dashboard 진입
```

**장점**: 비밀번호가 유출되더라도 이메일 접근 권한이 없으면 로그인 불가 → 보안 ↑

### 3-5. 로그인이 안 될 때 — 3초 진단

| 증상 | 가능한 원인 | 해결책 |
|------|-------------|-------|
| "Incorrect password" | 비밀번호 오타 또는 대소문자 | Caps Lock 확인, 비밀번호 재확인 |
| "Connection error" | 서버 다운 또는 네트워크 끊김 | Wi-Fi 확인, 30초 후 재시도 |
| OTP가 안 와요 | 스팸함에 있거나 이메일 지연 | 스팸함 확인 → 1분 기다리기 → Back to login 눌러 재발송 |
| "Invalid verification code" | 코드 오타 또는 만료 (보통 10분) | 처음부터 다시 시도 |
| 비밀번호를 잊었어요 | — | `Forgot password?` 링크 클릭 → 이메일로 재설정 링크 수신 |

## 4. 관리자 Dashboard 이용 방법

### 4-1. 접속 경로

1. `https://seoulinkhealth.com/login` 접속
2. 관리자 비밀번호 입력
3. `/admin`으로 자동 이동

### 4-2. 화면 전체 구조

```
┌──────────────────────────────────────────────────────────┐
│  SEOULINKHEALTH                    [🔄 새로고침] [Sign Out]│
├──────────────────────────────────────────────────────────┤
│                                                          │
│  [📊 전체 신청]  [🆕 신규 신청]  [💬 전체 문의]  [🆕 신규 문의] │
│                                                          │
│  (📬 Applications)  (✉️ Inquiries)             [⬇ CSV 내보내기]│
│                                                          │
│  [🔍 검색창]                    [All] [New] [Reviewed] [Contacted]│
│                                                          │
│  ┌────────────────────────────────────────────────────┐ │
│  │ 이름 / 이메일 / 제목             날짜        상태  │ │ ← 클릭 시 상세 모달
│  └────────────────────────────────────────────────────┘ │
│  ┌────────────────────────────────────────────────────┐ │
│  │ ...                                                │ │
│  └────────────────────────────────────────────────────┘ │
│                                                          │
│                 [이전] 1 / 5 페이지 [다음]                │
└──────────────────────────────────────────────────────────┘
```

### 4-3. 기능별 설명

#### 📊 상단 통계 카드 (4개)
| 카드 | 의미 |
|------|------|
| 전체 신청 | Expert Network에 지원한 모든 전문가 |
| 신규 신청 | 아직 "New" 상태로 검토 대기 중인 건 |
| 전체 문의 | Request Form으로 들어온 모든 자문 요청 |
| 신규 문의 | 아직 "New" 상태로 답변 대기 중인 건 |

파란 숫자로 **"N unread"** 표시가 뜨면 "아직 처리 안 된 건수"를 뜻합니다.

#### 📑 두 개의 탭

- **Applications (지원서)**: `/network`에서 전문가가 Application Form 제출한 기록
- **Inquiries (문의)**: `/request`에서 기업이 Inquiry Form 제출한 기록

탭 전환 시 필터와 페이지는 초기화됩니다.

#### 🔍 검색 & 필터

- **검색**: 이름, 이메일, 제목, 회사 등 아무 키워드 입력 → 타이핑 후 **0.3초 뒤 자동 검색** (디바운스)
- **상태 필터**: `All` / `New` / `Reviewed` / `Contacted` 네 버튼 중 하나 선택

#### 👁 상세 모달

카드 전체 클릭 → 오른쪽에서 모달이 뜹니다. 여기서:
- 지원자/문의자의 **전체 정보**(연락처, 경력, 문의 내용 등) 확인
- **상태 변경** 드롭다운: `New → Reviewed → Contacted` 순서로 진행하는 게 일반적
- 상태 변경 시 초록색 토스트로 "상태가 변경되었습니다" 확인

#### 📥 CSV 내보내기

- 우측 상단 **CSV** 버튼 클릭
- 현재 탭의 현재 페이지 데이터를 엑셀로 열 수 있는 CSV 파일로 다운로드
- 파일명 예시: `application_2026-04-22.csv`

#### 🔁 자동 새로고침

- **30초마다 자동**으로 최신 데이터를 가져옵니다
- 즉시 갱신하고 싶으면 상단 **🔄 새로고침** 버튼

#### 📄 페이지네이션

한 페이지당 10건씩 표시. 하단에 `이전 / N / M 페이지 / 다음` 컨트롤.

### 4-4. 추천 운영 루틴

**매일 아침 루틴 (5분)**:
1. `/login` 접속
2. 상단 "신규 신청 N", "신규 문의 N" 숫자 확인
3. Inquiries 탭 → `New` 필터 → 각 건 클릭 → 확인 후 `Reviewed`로 변경
4. 담당 전문가 배정 후 연락 → `Contacted`로 변경

**주 1회 루틴 (15분)**:
- CSV 내보내기 → 월간 통계 집계
- `Contacted` 건 중 답변 완료된 것은 종료 처리 (별도 메모)

## 5. 회사가 로그인했을 때 보여지는 화면과 기능

### 5-1. 로그인 흐름 요약

```
[/company/register]  ─────→  계정 생성
         │
         ↓
[/company/login]     ─────→  이메일 + 비밀번호
         │
         ↓
  [이메일로 OTP 수신]  ─→  6자리 코드 입력
         │
         ↓
[/company/dashboard] ─────→  본격 이용 시작
```

### 5-2. Dashboard 진입 시 보이는 화면

상단에 검은 네이비 헤더(SEOULINKHEALTH 로고 + 회사명/담당자명 + Sign Out 버튼)가 있고, 그 아래 **세 개의 탭**이 있습니다.

```
┌──────────────────────────────────────────────────────────┐
│  SEOULINKHEALTH       [회사명]  [담당자]    [Sign Out]   │
│  K-HEALTH BUSINESS PLATFORM                              │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  [ Overview ]  [ My Inquiries ]  [ Settings ]            │
│                                                          │
│  ↓ 선택된 탭 내용                                         │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 5-3. Overview 탭 — "내 문의 한눈에"

4개의 **통계 카드** + **최근 5건 목록**:

| 통계 카드 | 의미 | 색 |
|-----------|------|------|
| Total Inquiries | 내가 보낸 전체 문의 수 | 네이비 |
| Open | 아직 답변 대기 중 | 🔵 파랑 |
| Answered | 전문가 답변 완료 | 🟢 초록 |
| Closed | 상담 종료 | ⚪️ 회색 |

그 아래 **Recent Inquiries** 박스 — 최근 5개 문의만 빠르게 확인.
각 항목: 문의 제목 / 작성일 / 상태 배지.

### 5-4. My Inquiries 탭 — "내 문의 전부 보기"

내가 제출한 **모든 문의 목록**. 각 항목:
- 문의 제목
- 작성일 + 마지막 업데이트 날짜
- 현재 상태 배지 (Open / Answered / Closed)

상단 우측에 `Open Q&A` 버튼 → 문의 Q&A 페이지로 이동(답변 주고받기).

### 5-5. Settings 탭 — "내 계정 관리"

두 개의 하위 섹션:

#### 🔧 Profile Settings
- **Company Name**: 회사명
- **Contact Person**: 담당자 이름
- **Telephone**: 연락처

수정 후 `Save Changes` 클릭. 성공 시 초록 토스트 알림.

#### 🔐 Change Password
- **Current Password**: 기존 비밀번호
- **New Password**: 새 비밀번호 (최소 8자)
- **Confirm New Password**: 새 비밀번호 재입력

두 번째와 세 번째가 일치해야 변경됩니다. 변경 후에는 다음 로그인 시 새 비밀번호 사용.

### 5-6. 회사 Dashboard 이용 시나리오

**가입 & 첫 사용 (3분)**:
1. `/company/register`에서 회사 정보 + 담당자 + 이메일 + 비밀번호 입력
2. 이메일 인증
3. `/company/login`에서 로그인 → OTP 입력
4. **자동으로 Dashboard 진입** → Overview 탭에서 "No inquiries yet." 확인

**문의 제출 & 추적 (10분)**:
1. `/request` 이동 → Inquiry Form 작성
2. 제출 직후 Dashboard로 돌아오면 **Total Inquiries = 1**, Overview에 해당 건 표시
3. 상태가 **Open**인 동안 답변 기다림
4. 전문가가 답변 달면 상태가 **Answered**로 변경 → 이메일 알림
5. 상담이 마무리되면 관리자가 **Closed** 처리

## 6. Dashboard 개념 이해하기 (초보자용)

### 6-1. Dashboard, 한 문장으로 말하면?

> **"내 업무 현황을 한눈에 보여주는 상황판"**

자동차 계기판(대시보드)을 떠올리면 쉽습니다. 속도, 연료, 엔진 상태를 따로따로 보지 않아도 운전석 앞에서 **한눈에** 볼 수 있죠.

### 6-2. 현실 속 비유

#### 🏠 집안 현관의 "알림판"
- 오늘 택배 2건 도착 예정
- 가족 일정 3건
- 미확인 우편물 1건

현관 앞에서 이 보드만 봐도 "오늘 뭘 신경써야 하지?"가 바로 파악됩니다.

#### 🍳 식당 주방의 "주문 보드"
- 들어온 주문: 12건
- 조리 중: 4건
- 완료 대기: 2건

쉐프는 이 보드만 봐도 주방이 잘 돌아가는지, 밀리는 곳이 있는지 알 수 있습니다.

### 6-3. 왜 Dashboard가 필요한가요?

**Dashboard가 없을 때**:
- 내가 보낸 문의가 몇 건인지 모름
- 답변이 왔는지 확인하려면 이메일을 전부 뒤져야 함
- "고객님, 요청 어떻게 되고 있나요?" 물으면 대답이 어려움

**Dashboard가 있을 때**:
- 로그인 → **3초 안에** 오늘 할 일 파악
- "Open 2건"을 보고 우선순위 바로 판단
- 누가 물어봐도 화면 하나만 보여주면 상황 설명 끝

## 8. 자주 묻는 질문 (FAQ)

**Q. 모바일에서도 쓸 수 있나요?**
A. 네. 반응형으로 설계되어 스마트폰, 태블릿에서도 동일한 기능을 사용할 수 있습니다.

**Q. 여러 직원이 같은 회사 계정을 쓸 수 있나요?**
A. 회사별 계정은 하나입니다. 로그인 정보를 공유하면 동일 Dashboard를 볼 수 있지만, **OTP가 등록 이메일로만 가기 때문에** 해당 이메일에 접근 권한 있는 직원만 실질적으로 로그인 가능합니다.

**Q. 관리자 비밀번호를 변경하려면?**
A. 서버 환경변수 `ADMIN_PASSWORD`를 업데이트하고 백엔드를 재시작하면 됩니다. (개발자 도움 필요)

**Q. GCP 말고 다른 클라우드를 써도 되나요?**
A. 네. 현재 기본 설정은 Vercel(프론트) + Railway(백엔드) + Supabase(DB)로 최소 비용 구조입니다. AWS, Azure, Fly.io 등 어디든 배포 가능합니다.

**Q. 데이터는 어디에 저장되나요?**
A. PostgreSQL 데이터베이스에 저장됩니다. Prisma ORM으로 관리되며, 백업은 클라우드 DB의 자동 스냅샷 기능을 이용합니다.

**Q. 이메일이 발송되지 않아요.**
A. 백엔드의 SMTP 설정(`SMTP_HOST`, `SMTP_USER`, `SMTP_PASS`)을 확인해야 합니다. SendGrid나 Gmail SMTP 등을 쓸 수 있습니다.

**Q. 숫자가 자동으로 업데이트되나요?**
A. 관리자 Dashboard는 **30초마다 자동 갱신**됩니다. 회사 Dashboard는 탭 이동 또는 페이지 새로고침 시 갱신됩니다.
